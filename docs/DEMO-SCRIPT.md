# 测评演示讲稿

## 1. 开场

这个作品是一个面向大学生求职场景的申请管理看板。我希望解决的不是单纯的“记录岗位”，而是帮助用户在多岗位并行申请时，始终知道自己投了什么、做到哪一步、下一步该做什么。

## 2. 用户痛点

我先聚焦了几个最常见的问题：

- 多个截止日期容易遗漏
- 不同岗位材料要求不同，状态容易混乱
- 面试安排和 follow-up 节奏难统一管理
- 职位下线后，面试前找不到原始 JD

## 3. 页面结构

我把整个产品拆成五块：

- 首页总览，帮助用户快速了解整体进度
- 行动中心，把优先处理和待跟进事项提出来
- 看板区，用来管理申请流程推进
- 时间线，用来查看临近节点
- 岗位详情，用来沉淀 JD、材料版本、联系人和流程记录

## 4. 设计亮点

相比普通表格或清单，我重点补了几个更接近真实求职场景的能力：

- JD 快照，避免岗位下线后找不到要求
- 简历版本绑定，减少“一份简历投所有岗位”的问题
- Follow-up 管理，把联系人和跟进时间纳入流程
- 阶段行动建议，告诉用户下一步该做什么

## 5. 技术实现

技术上我没有堆复杂框架，而是用了轻量全栈方案：

- 前端用原生 HTML、CSS、JavaScript
- 后端用 Node 原生 HTTP 服务
- 数据存储在本地 JSON 文件

这样做的好处是结构清晰、容易展示，同时又保留了真实的接口和持久化能力。

## 6. 收尾

整体来说，这个作品希望体现两个点：

- 第一，它能真实帮助学生管理求职申请过程
- 第二，它不是停留在概念说明，而是已经做成了可以运行、可以交互、可以保存数据的完整作品
